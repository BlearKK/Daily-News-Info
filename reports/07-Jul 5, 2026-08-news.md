<h4 id="aiml">AI/ML &amp; 软件工程</h4>
<ul>
<li><strong><a href="https://simonwillison.net/2026/Jul/4/better-models-worse-tools/#atom-everything">Better Models: Worse Tools (模型越强，工具越难用？)</a></strong></li>
<li><strong>摘要</strong>：著名开发者 Armin 发现，在最新版的 Claude 模型（如 Opus 4.8 和 Sonnet 5）中，在使用自定义编辑工具时，相较于旧模型反而更容易生成不符合 Schema 的畸形调用。</li>
<li><strong>要点</strong>：<ul>
<li><strong>过度拟合</strong>：新模型因针对 Claude Code 的内置编辑工具（Search/Replace）进行了高强度强化训练（RL），导致泛化到第三方定制工具（如 Pi）时兼容性变差。</li>
<li><strong>设计冲突</strong>：OpenAI 的 Codex 使用 <code>apply_patch</code> 机制，而 Anthropic 采用 <code>str_replace</code>，平台工具分化给多模型适配带来挑战。</li>
<li><strong>应对策略</strong>：第三方编码助手未来可能需要根据底层模型的偏好，动态提供多套编辑工具 Schema。</li></ul></li>
<li><strong>标签</strong>：#AI #LLM #Claude #软件工程</li>
<li><strong>来源与时间</strong>：Simon Willison · 1 天前</li>
</ul>
